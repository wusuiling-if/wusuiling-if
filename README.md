# Han Shengyi

I build small AI systems around agents, memory, RAG, and product prototypes.

My recent work is mostly about turning fuzzy AI-product ideas into running systems: agent loops, tool interfaces, memory services, API gateways, evaluation surfaces, and web dashboards that make model behavior easier to inspect.

## Currently Exploring

- Agent workflows with explicit state, routing, tool use, and observable failure cases.
- Long-running conversation systems with memory retrieval, summarization, and preference adaptation.
- LLM application infrastructure: OpenAI-compatible APIs, streaming, auth, quotas, logs, routing, and admin panels.
- Fast product prototyping with React, TypeScript, FastAPI, Node.js, and AI coding tools.
- Applied multimodal models: gaze estimation, object detection, pose/keypoint extraction, and lightweight CV pipelines.

## Notes

I care about AI products that are not only impressive in a demo, but inspectable when they fail.  
Most of my prototypes include some combination of state visibility, logs, debug panels, evaluation hooks, or explicit failure-case taxonomy.

Open to AI application engineering, AI product engineering, and agent-related roles.
